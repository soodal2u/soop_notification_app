# 코드 설명

## .github/workflows/release.yml

- **1~7번 줄**: 워크플로우의 이름과 실행 트리거를 설정합니다. `main` 브랜치에 푸시되거나 `v*` 형태의 태그가 푸시될 때 작동합니다.
- **9~10번 줄**: 워크플로우가 리포지토리에 쓰기 권한(Release 생성 등)을 가질 수 있도록 권한을 설정합니다.
- **15~17번 줄**: 소스 코드를 체크아웃합니다.
- **19~23번 줄**: 자바 환경(JDK 17)을 설정합니다. 안드로이드 빌드에 필요합니다.
- **25~29번 줄**: Flutter 환경을 설정합니다. `pubspec.yaml`에 명시된 SDK 버전과 호환되도록 설정되었습니다.
- **31~36번 줄**: `pubspec.yaml` 파일에서 `version` 정보를 추출하여 환경 변수(`VERSION`, `TAG_NAME`)에 저장합니다. 이를 통해 앱 버전과 동일한 태그로 릴리즈를 생성합니다.
- **38~39번 줄**: 프로젝트의 의존성 패키지들을 설치합니다.
- **41~42번 줄**: Release 모드로 APK를 빌드합니다.
- **44~50번 줄**: `assets/changelog.json` 파일에서 가장 최신 버전의 변경 사항들을 추출하여 릴리즈 노트(`body`)에 사용하기 위해 환경 변수에 저장합니다. `jq`를 사용하여 JSON을 파싱합니다.
- **52~65번 줄**: `softprops/action-gh-release` 액션을 사용하여 GitHub Release를 생성합니다. 빌드된 APK 파일을 업로드하고, 추출한 버전명을 태그와 타이틀로 사용합니다.
